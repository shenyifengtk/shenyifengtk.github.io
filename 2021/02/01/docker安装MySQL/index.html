docker安装MySQL

发表于 2021-02-01 | 分类于 docker | 评论数： | 阅读次数：
本文字数： 4.8k | 阅读时长 ≈ 4 分钟

都2021了，学会用docker 来构建软件开发吧，快速构建环境，容器比虚拟机更加轻量化，拥有诸多好处。下面我使用docker安装运行MySQL例子，为大家演示docker使用方法。

搜索镜像

docker search mysql

这些都是镜像都是dockerhub官方镜像，如果想了解它们详细信息，可以去官网搜索关键字。这里我选择第一个，MySQL官方镜像。

拉取镜像

docker pull mysql

这里还可以选择MySQL版本

docker pull [选项] [Docker Registry地址]<仓库名>:<标签>

  • Docker Registry地址： 地址的格式一般是 <域名/IP>[:端口号] 。 默认地址是 Docker Hub。

  • 仓库名： 如之前所说， 这里的仓库名是两段式名称， 既 <用户名>/<软件名> 。 对于 Docker Hub， 如果不给出用户名， 则默认为 library ， 也就是官方镜像。

  • 标签: 如果不指定tag(类似:8.0)，那么docker默认会为我们加上latest

    运行镜像

    简单运行镜像

    docker run –name mysql-name -e MYSQL_ROOT_PASSWORD=password -d mysql:tag

  • name : mysql镜像运行容器名称

  • d : 守护进程方式运行镜像

  • e : 镜像系统变量设置，如上面例子MYSQL_ROOT_PASSWORD 设置MySQL root密码 ，详情变量可以查看镜像说明。

  • tag: 指定运行image标签，不写默认就运行laster，像MySQL这类镜像tag 多为版本号，支持多个版本MySQL运行。

还可以通过-p 网络映射，将容器内端口映射到运行主机上某一个端口，这个我们访问这个端口，就可以连接到容器内MySQL了。

docker run –name mysql-name -p 3306:3306 -e MYSQL_ROOT_PASSWORD=password -d mysql:tag

将容器内3306端口映射到主机上3306，使用方式 -p 主机端口:容器端口

MySQL容器数据保存

在MySQL容器的数据一般有两个存储方式。

  • 让Docker通过使用其自身的内部卷管理将数据库文件写入主机系统上的磁盘来管理数据库数据的存储，这种是默认存储方式，但是有个弊端，数据会随着容器生命周期结束而被销毁，不符合一个数据库保存数据标准。
  • 在容器运行前，通过数据卷映射，将MySQL数据保存到主机路径上。当容器被停止时，数据会保存到主机路径上的，这种方式更加符合MySQL日常使用。

    docker run –name mysql-name -p 3306:3306l -v /my/own/datadir:/var/lib/mysql -e MYSQL_ROOT_PASSWORD=password -d mysql:tag

-v 将主机路径/my/own/datadir 路径挂载到容器内/var/lib/mysql路径，这样在/var/lib/mysql下保存MySQL data 就会保存到主机/my/own/datadir 下，当容器再次启动时MySQL就会从/my/own/datadir 加载数据。前提是要保证/my/own/datadir 目录文件时存在的。如果不知道MySQL容器保存数据在哪里，可以进入容器查看配置文件

进入容器

docker exec -it mysql-name /bin/bash

查看配置文件

cat /etc/mysql/my.cnf

数据保存到/var/lib/mysql 路径下,exit 退出，将容器data 路径下文件copy到主机下。

docker cp mysql-name:/var/lib/mysql/ /my/own/datadir
